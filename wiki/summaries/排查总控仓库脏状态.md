# 排查总控仓库脏状态

用 OpenAI Agents SDK 设计"仓库健康度审计 Agent 系统"，对 d:\xiangmu（27 GB / 17000+ 文件）进行自动化清理。

## 问题现状

| 指标 | 值 |
|------|-----|
| 总路径 | d:\xiangmu |
| 总大小 | 27 GB |
| 文件数 | 17000+ |
| 已纳入 _kb | 少量 |
| .gitignore 覆盖率 | 未知 |
| 过期依赖 | 未知 |
| 重复/垃圾文件 | 未知 |

## 解决方案：3 Agent 流水线

用 OpenAI Agents SDK 的 Handoff 模式构建 3 个专业 Agent，按阶段流水线处理：

```
用户启动 → [文件分类Agent] ──handoff──→ [依赖分析Agent] ──handoff──→ [决策Agent] → 输出报告
```

### Agent 设计

#### Phase 1: 文件分类 Agent（FileClassifier）

```python
file_classifier = Agent(
    name="文件分类专家",
    instructions="""你是一个文件分类专家。对给定目录中的文件进行分类：
    1. .gitignore 命中（应忽略）
    2. 数据文件（.csv / .db / .json 数据）
    3. 代码文件（.py / .ts / .html）
    4. 文档文件（.md / .txt / .docx）
    5. 垃圾/临时文件（可删除）
    6. 未知类型（需人工判断）
    输出 JSON 格式的分类结果。""",
    model="gpt-5-nano",
    tools=[scan_directory, check_gitignore, get_file_stats],
    output_type=ClassificationReport,
)
```

#### Phase 2: 依赖分析 Agent（DependencyAnalyzer）

```python
dep_analyzer = Agent(
    name="依赖分析专家",
    instructions="""分析项目依赖：
    1. 检查各项目 requirements.txt / package.json 的版本
    2. 识别过期依赖（对比 PyPI / npm 最新版）
    3. 扫描已知安全漏洞（pip-audit / npm audit）
    4. 生成更新建议（含风险等级）""",
    model="gpt-5-nano",
    tools=[read_requirements, check_pypi_version, run_pip_audit],
    output_type=DependencyReport,
)
```

#### Phase 3: 决策 Agent（DecisionMaker）

```python
decision_maker = Agent(
    name="清理决策专家",
    instructions="""根据文件分类和依赖分析结果，生成清理建议：
    1. 保留：核心代码/文档/配置
    2. 归档：历史版本/旧数据
    3. 删除：垃圾/临时/重复文件
    4. 更新：过期依赖（含回滚方案）
    每条建议附带理由和风险评估。""",
    model="gpt-5",
    tools=[estimate_size, check_git_history],
    output_type=CleanupPlan,
)
```

#### 分诊 Agent（Router）

```python
router = Agent(
    name="仓库审计调度台",
    instructions="协调三个专家完成仓库健康度审计，按顺序执行。",
    handoffs=[file_classifier, dep_analyzer, decision_maker]
)
```

## 安全护栏设计

```python
# 输入护栏：防止误删关键文件
@input_guardrail
async def no_critical_delete(ctx, agent, input):
    """阻止删除包含 .git / AGENTS.md / config 的目录"""
    critical_patterns = ['.git', 'AGENTS.md', 'config.yaml', '.env']
    for pattern in critical_patterns:
        if pattern in str(input):
            return GuardrailFunctionOutput(
                tripwire_triggered=True,
                output_info=f"拒绝操作：涉及关键文件 {pattern}"
            )
    return GuardrailFunctionOutput(tripwire_triggered=False)

# 输出护栏：确保清理建议可回滚
@output_guardrail
async def must_be_reversible(ctx, agent, output):
    """确保所有删除建议都有备份方案"""
    plan = output
    for item in plan.get("delete", []):
        if not item.get("backup_path"):
            return GuardrailFunctionOutput(
                tripwire_triggered=True,
                output_info=f"删除建议缺少备份路径: {item['path']}"
            )
    return GuardrailFunctionOutput(tripwire_triggered=False)
```

## 渐进式执行策略

遵循 [[concepts/渐进式开发]] 原则：

1. **先扫描，不动文件** — Phase 1-2 只读操作
2. **人工审核后再删** — Phase 3 的删除操作需用户确认
3. **git tag 标记** — 每个 Phase 完成后打 tag
4. **可回退** — 所有删除操作先备份到 .archive/
