# 交付项目管理

> 本工作区交付项目（投标/工程/咨询）的本地工作流总结，涵盖 SuanJia 三类项目（neimeng/huaying-labor/chengdu-fire）、zhaobiaowenjian 4 Phase 招投标流程、xunbiao 招标情报采集。

## 当前工具链

### SuanJia 工程项目体系（3 个活跃交付项目）
- **目录**：`SuanJia/projects/`，每个项目独立子目录
- **统一元数据**：每个项目 `project.json`（JSON Schema 模板 `projects/_templates/project-template.json`）
- **项目状态**：
  - `neimeng/`（内蒙厂区道路投标，260513 人工修正报价表）
  - `huaying-labor/`（华蓥体育公园栽植劳务，project.json + analysis.json + risk-report.md）
  - `chengdu-fire/`（宜宾宁德时代消防劳务，谈判准备阶段，negotiation-guide.md + meeting-prep.md）
- **项目文件标准**：
  - 报价：`附件1：投标报价文件（厂区道路工程）.xlsx`
  - 成本：cost_calculation.py / cost_final.py
  - 风险：risk-report.md
  - 谈判：negotiation-guide.md / meeting-prep.md
  - 图纸：`图纸/` + `图纸_图片/` + `定额摘要/`
  - 归档：`_archive/` 旧版代码
- **生命周期**：见 `projects/README.md`（生命周期/目录结构/字段说明/命名规则）

### zhaobiaowenjian 4 Phase 招投标流程（核心方法论）
- **Phase 0 初筛**（10 分钟）：6 项自动检查
  - 中小微企业 / 特定资格 / 经营范围 / 准备时间 / 保证金 / 禁止情形
  - 全部通过 → Phase 1，有不通过 → 告知原因
- **Phase 1 多维分析**（5 维并行）：资质/经济/技术/风险/竞争
- **Phase 2 成本定价**：模板自动选择
  - 材料采购：材料价+运费+措施费+质保金+管理费+代理费
  - 工程劳务：人工+材料+机械+措施+管理+安全+代理费
  - 定价策略：保守 10-13% / 推荐 13-17% / 激进 17-20%
- **Phase 3 投标准备**：T-7 → T-5 → T-3 → T-1 → T 日时间节点
- **Phase 4 复盘归档**：中标（缴费/签合同/执行） / 未中标（分析原因/退保证金/归档）

### xunbiao 招标情报采集
- **数据源**：广安市公共资源交易网 API（`https://www.gasggzy.cn`）
- **采集方式**：GitHub Actions cron（每日 09:00 / 18:00 北京时间）
- **本地版**：`v1/cloud_main.py` + `v1/feishu_push.py`
- **Web 版**：`web/`（Vue 3 + Vant 4 移动端）
- **飞书推送**：华蓥招标情报 → 群机器人 Webhook

### SuanJia 经营部 16 工具（项目交付相关）
- `add_project` / `update_project` / `search_projects` / `get_project_summary`
- `approve_project` / `analyze_bid` / `generate_bid_document` / `get_bid_checklist`
- `assess_risk` / `submit_quote` / `accept_project`
- **project_pipeline.json**：投标流程跟踪（前期跟踪 → 招标中 → 已中标）

### SuanJia 夜间自动化任务系统（night-tasks/）
- `orchestrator.py`：按 batch 执行子任务并收集结果
- `batch_runner.py`：循环调用 master_loop.py
- `master_loop.py`：调用 Claude Code CLI 执行 pending 任务
- `auto_runner.py` / `auto_runner_v2.py`：批量 prompt 依次执行
- 5 个 batch 已完成（batch1.json / batch3_report.md / final_report.md）

## 痛点

1. **模板化程度不均**：huaying-labor 有完整 project.json + analysis.json + risk-report.md；chengdu-fire 只有谈判文档；neimeng 用 Python 脚本而非 JSON 元数据
2. **跨项目复用难**：3 个项目共享 `_archive/` 模式但未建共享成本库；每个项目手写 cost_calculation.py
3. **4 Phase 流程未代码化**：zhaobiaowenjian 4 Phase 是 CLAUDE.md 自然语言描述，需人工按节点执行
4. **招投标与项目交付割裂**：xunbiao 采集 → 人工筛 → 经营部 add_project，无自动串联
5. **进度跟踪依赖 WORKLOG**：项目进度靠 WORKLOG.md 散落记录，无独立里程碑跟踪
6. **风险评估静态**：risk_registry.json 有但 risk_assessment 写在 projects.json 内（PJ004），两处数据可能不一致
7. **复盘归档缺失**：Phase 4 复盘（中标/未中标原因分析）无结构化模板
8. **夜间任务结果散落**：night-tasks/results/ 多个 .jsonl + .log 混合，难追踪任务状态
9. **多 Agent 接力未启用**：5 Phase 流程（尤其 Phase 1 多维分析）天然适合并行，现状单 Agent 串行

## 优化方向（结合 AI Agent）

### 1. 项目元数据标准化（建模板库）
- **方案**：用 `projects/_templates/project-template.json` 强约束所有新建项目
- **CI 校验**：项目根必须有 project.json + 必填字段（id/name/client/budget/stage/deadline/risks）
- **AI 角色**：Codex 写模板，Trae 迁移现有 3 个项目到模板

### 2. 4 Phase 流程代码化（biz_mcp 工具增强）
- **Phase 0**：`biz_mcp.analyze_bid` 自动执行 6 项检查 → 输出 Pass/Fail + 原因
- **Phase 1**：5 维并行（5 智能体或 5 sub-agent）
  - 资质子 Agent → 搜索公司库 + 公开资质
  - 经济子 Agent → 成本计算 + 历史项目对比
  - 技术子 Agent → 解析图纸 PDF + 工程量清单
  - 风险子 Agent → risk_registry 匹配 + LLM 评估
  - 竞争子 Agent → 公开招标记录 + 对手分析
- **Phase 2**：`cost_mcp.create_cost_estimate` 自动按模板（材料/劳务）计算 + 利润率建议
- **Phase 3**：`generate_bid_document` 自动生成投标文件 + 时间节点提醒（Hermes Cron）
- **Phase 4**：中标/未中标 → 自动建复盘文档 + 风险库更新

### 3. xunbiao → 经营部 → 项目 自动化串联
- **流程**：xunbiao 抓招标 → 触发 Phase 0 初筛 → 通过则 `biz_mcp.add_project` → 飞书通知
- **AI 角色**：
  - xunbiao：GitHub Actions 抓取 + Claude Code 评估
  - 经营部：飞书群消息触发 add_project
  - 飞书推送：Hermes Cron 每日汇总新项目

### 4. 进度跟踪系统（PROJECT_STATUS + project_pipeline）
- **现状**：每个项目 PROJECT_STATUS.md 独立维护
- **升级**：建项目总览仪表盘（`d:\xiangmu\信息\项目仪表盘.md`）
  - 实时拉取所有项目 PROJECT_STATUS.md 的"当前进展"段
  - 按 stage 排序：前期跟踪 / 招标中 / 已中标 / 执行中 / 已完成
  - 飞书每日推送给项目相关人
- **AI 角色**：Claude Code 知识管家负责聚合 + Hermes Cron 触发

### 5. 复盘知识库（中标/未中标原因结构化）
- **方案**：建 `SuanJia/projects/_retros/` 目录，存 `项目名-复盘.md`
- **必填字段**：中标价 vs 预算 / 利润率 vs 预期 / 风险预案 vs 实际 / 学习要点
- **AI 角色**：Claude Code 自动生成初稿 → 用户审阅
- **价值**：跨项目横向学习（哪些项目类型利润率最高/哪些甲方最准时付款）

### 6. night-tasks 升级（5 智能体并行）
- **现状**：Claude Code CLI 单进程串行
- **升级**：
  - orchestrator.py 调度 5 智能体（Trae/Codex/OpenCode/Hermes/Claude Code）
  - 每个 batch 内任务可并行（IO 密集 vs CPU 密集分类）
  - 结果统一汇总到 results/final_report.md
- **AI 角色**：Claude Code 编排 + 各智能体执行

### 7. 风险预警系统
- **触发条件**：客户付款延迟 > X 天 / 项目超期 > Y 天 / 风险等级变高
- **实现**：
  - Hermes Cron 每日检查 project_pipeline.json + 飞书消息关键词
  - 触发 → `biz_mcp.assess_risk` 重评估 → 飞书 @相关人
- **AI 角色**：Hermes 定时 + Claude Code 评估
