# 客户管理实践

> 本工作区客户管理（CRM）的本地工作流总结，涵盖 SuanJia 经营部、zhaobiaowenjian 招标工作流、信息域飞书用户映射三个数据源。

## 当前工具链

### SuanJia 经营部（主 CRM 中心）
- **数据文件**（`d:\xiangmu\SuanJia\departments\01-yingying-bu\data\`）
  - `clients.json` — 客户库（id/name/client_type/tier/contact_person/phone/email/region/projects/notes）
  - `communications.json` — 沟通记录库（按用户/时间/角色）
  - `projects.json` — 项目库（关联客户 id）
  - `project_pipeline.json` — 项目管道（投标流程跟踪）
  - `risk_registry.json` — 风险登记簿
- **MCP 工具**（`biz_mcp.py` 16 个工具）
  - `add_client` / `update_client` / `search_clients` / `get_client_summary`
  - `record_communication` — 沟通记录
  - `analyze_bid` / `assess_risk` — 风险评估
- **评分匹配**（zhaobiaowenjian/CLAUDE.md）
  - 资质 40% + 经营范围 25% + 中小微 15% + 人员 15% + 业绩 5%
  - 收到招标文件 → 自动从公司库匹配最合适公司

### zhaobiaowenjian 招标库
- **目录**：`00_公司资料/公司库.md`（四川展蓝建筑有限责任公司入库）
- **自动流程**：说"加公司：XXX"→ 智能体搜资质（住建部/信用中国/天眼查）→ 录入 → 用户确认
- **当前在库**：四川展蓝建筑（施工劳务不分等级）

### 信息域 飞书用户映射
- **变量**：`FEISHU_USER_MAP`（位置：`SuanJia/feishu_bridge/.env`）
- **用途**：将飞书用户 ID 映射为内部标识（经营部 16 工具 + 飞书桥接共用）

## 痛点

1. **客户数据稀疏**：clients.json 仅 1 条完整记录（CL001 邻水县华盛），其余客户 contact_person/phone/email 全空
2. **沟通记录空白**：communications.json 数组为空（`last_updated: 2026-05-13`），客户触达无据可查
3. **多源割裂**：经营部 clients.json / zhaobiaowenjian 公司库.md / 飞书 FEISHU_USER_MAP 三处独立维护，缺少全局客户视图
4. **人工录入依赖**：`source: 人工录入` 字段说明入库全靠手动，转化率低
5. **分级机制未用**：tier 字段（A/B/C）有定义但仅 1 条 C 级，无分级跟进策略
6. **无跟进提醒**：无 cron / Task Scheduler 触发客户回访，靠人工记
7. **招投标分析脱钩**：经营部 MCP 16 工具与 zhaobiaowenjian 4 Phase 分析流程无自动串联

## 优化方向（结合 AI Agent）

### 1. 全局客户视图（多源融合）
- **方案**：建 `_kb_personal/03-私有/客户库/`，单点维护，反向同步到 SuanJia clients.json + zhaobiaowenjian 公司库 + 飞书 FEISHU_USER_MAP
- **AI 角色**：Claude Code 知识管家负责 ETL（增量同步、冲突合并），Trae 负责日常录入
- **数据源**：
  - 飞书群消息 → `lark-im` 自动抽取客户名/联系方式
  - 招标文件 → `biz_mcp.analyze_bid` 自动填充客户信息
  - 公开数据 → Tavily 搜索企业资质自动补全

### 2. 沟通记录自动化
- **方案**：飞书桥接（feishu_bridge）每条入站/出站消息 → 调用 `biz_mcp.record_communication`
- **关键字段**：客户名、时间、议题、承诺事项、跟进日期
- **AI 角色**：Claude Code 解析对话提取结构化记录，Hermes Cron 每日汇总推送

### 3. 智能匹配引擎升级
- **现状**：5 维评分（资质/经营/中小微/人员/业绩）硬编码权重
- **升级**：用 LLM 评估非结构化信息（"曾总报价偏高需谈价"等业务语言）
- **AI 角色**：Claude Sonnet 做语义匹配 + 历史项目数据微调
- **预期**：匹配准确度从 60% → 85%+

### 4. 客户分级 + 跟进自动化
- **A 级**（战略客户）：每周自动生成跟进报告（lark-mail/lark-im 推送）
- **B 级**（重要客户）：每月一次 + 招投标节点提醒
- **C 级**（一般客户）：季度回访，按需响应
- **实现**：Hermes Cron 触发 + biz_mcp 触发提醒

### 5. 招投标→客户入库全链路
- **Phase 0**（初筛）：xunbiao 招标抓取 → `biz_mcp.search_clients` 反查 → 缺失则触发"加公司"流程
- **Phase 1**（分析）：5 维并行分析 → 匹配公司 → 评分
- **Phase 4**（复盘）：中标 → 自动升级 tier + 建 project_pipeline
- **AI 角色**：5 智能体并行调研（Trae/Codex/OpenCode/Hermes/Claude Code），Claude Code 汇总

### 6. 多 Agent 协作接入（参考 5 智能体验证结果）
- **Trae**：日常录入和快速查询（IDE 工具丰富）
- **Codex**：复杂匹配逻辑（GPT 规划强）
- **OpenCode**：免费额度兜底批量入库
- **Hermes**：定时跟进提醒（Cron 原生）
- **Claude Code**：知识管家治理（数据 ETL + 冲突合并）
