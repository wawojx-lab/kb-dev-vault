# 文档生产实践

> 本工作区文档生产（设计文档/技术规格/招标投标文件/简历/知识笔记）的本地工作流总结，涵盖 SuanJia 设计文档体系、信息域控制台、jianli 简历归档、_kb 知识库 4 类场景。

## 当前工具链

### SuanJia 设计文档体系（最完整）
- **目录**：`SuanJia/docs/`
- **结构**：
  - `design/` — 12 份设计文档（公司架构/数据总线/Agent 路由/各分部门设计）
  - `roles/` — 8 份角色定义（GM/秘书/各部门总监/顾问团）
  - `references/` — 参考资料（定额计算示例/宏业数据/二类厕所项目）
  - `meeting-minutes/` — 会议纪要（投资部讨论 v1）
  - `meeting-2026-05-20-agency-roster.md` — 全员大会纪要
  - `mistakes-log.md` — 错误日志
  - `gstacck-review-report-2026-05-20-round1/2.md` — 审查报告
- **命名规范**：
  - 设计文档：`XX-模块名-design.md`（如 `00-company-architecture.md`）
  - 角色定义：`role-name.md`（如 `gm.md`）
  - 会议纪要：`meeting-YYYY-MM-DD-主题.md`
  - 审查报告：`{agent}ack-review-report-YYYY-MM-DD-round{N}.md`
- **文档模板**：`docs/directory-standards.md`（目录命名规范）

## 痛点

1. **SuanJia docs 数量膨胀**：300+ 份文档（结构/角色/会议/审查/历史），找不到入口（docs/ 无 README 索引）
2. **文档版本混乱**：设计文档 v1/v2/final/round2 共存（`feishu-virtual-company-design-v1.md` + `v2.md` + `00-master-architecture-final.md`）
3. **结构不一致**：信息域 8 份 .md 高度结构化（编号 + 表 + wikilink），SuanJia docs 自由风格
4. **跨域引用缺失**：SuanJia docs 与 _kb wiki 互不引用（设计文档只引用 docs/ 内部）
5. **重复内容多**：gstack review 报告 round1/2 内容大量重复，未做 diff
6. **格式转换手动**：Markdown → 微信 HTML / PDF / Word 需手动或单文件脚本，缺批量
7. **机密分散**：API 密钥分散在 6+ 个 .env 文件（daily_stock_analysis/SuanJia/QingBao/9router/lianghua 等），密钥清单只列变量名，定位靠记忆
8. **简历无模板**：jianli/ 4 份简历结构各异（专业/通用/数据标注/卡特），投递时复用难
9. **历史归档无索引**：SuanJia `_archive/` 数百 MB 无清单，digital community `scripts/_tmp_we/` 重复 .asar
10. **会议纪要零散**：SuanJia docs/ 与 investment-dept/ 与 docs/meeting-minutes/ 三处散落
11. **错误日志无结构**：`mistakes-log.md` 是自由文本，无分类/严重度/状态
12. **跨智能体阅读体验差**：Claude Sonnet 读 .md 与 Hermes/Qwen 读 .md 表现差异大（已验证）

## 优化方向（结合 AI Agent）

### 1. SuanJia docs 建索引 + 版本管理
- **方案**：建 `docs/00-index.md`（按主题分类）+ `docs/_archive/.index.md`（归档索引）
- **版本管理**：用 Git tag（`v1.0`/`v2.0`/`final`）替代文件名带版本
- **AI 角色**：
  - Claude Code 知识管家：建索引 + 扫重复 + 生成 .index
  - 5 智能体并行扫不同子目录（Trae/Codex/OpenCode/Hermes/Claude Code）
  - Codex 规划重复文档合并策略

### 2. 信息域 → _kb 双向同步
- **方案**：信息域 .md 转 wiki pages
  - `03-API密钥清单.md` → `concepts/API-密钥管理`（脱敏后）
  - `06-MCP服务器.md` → `entities/MCP-体系`（MCP 列表为子节点）
  - `07-技能Skills.md` → `entities/gstack` + `entities/lark-cli`（拆分）
- **AI 角色**：Claude Code 知识管家 + flywheel `kb_ingest` 工具
- **收益**：单源维护（_kb 权威），信息域只展示不写

### 3. 设计文档协同编辑
- **方案**：SuanJia docs/design/ 迁移到飞书云文档（lark-doc 同步）
  - 多人协同编辑（飞书原生）
  - 评论/版本/历史（飞书原生）
  - AI 角色：Claude Code 定期 sync → 飞书；Trae 离线写 → push 飞书
- **优势**：手机可看、可评论、可@人

### 4. 招标文件 4 Phase 文档化
- **现状**：zhaobiaowenjian 4 Phase 在 CLAUDE.md 自然语言描述
- **升级**：建 4 份标准模板
  - `templates/phase0-初筛报告.md`
  - `templates/phase1-多维分析.md`（5 维 5 子表）
  - `templates/phase2-成本定价.md`
  - `templates/phase3-投标准备.md`（T-7/T-5/T-3/T-1/T 日时间节点表）
  - `templates/phase4-复盘.md`
- **AI 角色**：`biz_mcp.generate_bid_document` 自动套模板填充

### 5. Markdown → 多格式输出工具链
- **PDF**：`/make-pdf`（gstack，出版级 1in 边距/页码/水印）
- **微信 HTML**：`feishu_bridge/convert_reports.py`（已存在，需优化 CSS）
- **Word (.docx)**：lark-doc 导入（飞书原生）
- **飞书云文档**：lark-doc 创建
- **批量转换**：Claude Code 写 orchestrator（一次输入，输出 4 格式）
- **使用场景**：
  - 设计文档 → PDF（存档交付）
  - 分析报告 → 微信 HTML（手机看）
  - 招标文件 → Word（甲方提交）
  - 复盘文档 → 飞书云文档（团队协看）

### 6. 简历模板化（jianli 升级）
- **方案**：建 `_meta/templates/简历-通用.md` + `_meta/templates/简历-数据标注.md` + `_meta/templates/简历-卡特彼勒.md`
- **AI 角色**：
  - 写入时：用户说"投 XX 公司" → Claude Code 自动选模板 + 填充
  - 维护时：4 份简历 → 1 份 master + 3 份针对性裁剪
- **风险**：含个人信息，严格控制访问（_kb_personal/03-私有/）

### 7. 错误日志结构化
- **方案**：`mistakes-log.md` → 改用 frontmatter + 字段
  ```yaml
  ---
  type: concept
  title: 错误日志
  category: [部署/编码/沟通/...]
  severity: low/medium/high/critical
  status: open/fixing/closed
  ---
  ```
- **AI 角色**：
  - Claude Code 自动捕获（从 WORKLOG 关键词"踩坑"/"问题"/"修复"提取）
  - 飞书推送给 TraeSolo 审查

### 8. 跨域 wikilink 网络
- **方案**：SuanJia docs ↔ _kb wiki 双向 wikilink
  - SuanJia docs/04-siwei-bu-design.md ↔ _kb/wiki/entities/Claude-Code.md
  - SuanJia docs/01-yingyingbu-design.md ↔ _kb/wiki/entities/OKF.md
- **工具**：flywheel `kb_compile` 跨域分析
- **AI 角色**：Claude Code 知识管家（合规检查：脱敏后引用）

### 9. Obsidian 作 SuanJia docs 前端
- **方案**：把 SuanJia/docs/ 软链接到 _kb/wiki/_external/suanjia-docs/
- **价值**：Obsidian 全文搜索 + 反向链接 + 图谱视图
- **风险**：SuanJia docs 含敏感信息（公司架构/客户名），需脱敏或隔离 vault

### 10. 文档生产 AI 协作流程（标准化）
- **写**：用户口述 → Claude Code 起草 → TraeSolo 审查 → 用户确认
- **审**：5 智能体并行（不同视角：技术/产品/合规/UX/数据）
- **发**：自动分发到目标位置（飞书云文档/PDF/Markdown/HTML）
- **存**：自动归类到 _kb（OKF frontmatter） + Git 提交
