# 开发流程最佳实践

基于多项目 5 智能体接力 + 35 项目状态档案综合的开发流程方法论。

## 流程图（DAG 视角）

```
需求 → 调研 → 设计 → 任务拆解 → 接力计划
                          ↓
                  ┌───────┼───────┐
                  ↓       ↓       ↓
              智能体 A   智能体 B  智能体 C
                  ↓       ↓       ↓
                  └───────┼───────┘
                          ↓
                     集成测试 → 部署 → 监控
```

## 关键阶段

### 1. 需求
- 明确"做什么"和"不做什么"
- 验收标准（"完成"的定义）
- 优先级（必要 / 重要 / 锦上添花）

### 2. 调研
- 类似项目 / 库 / 框架
- 用户偏好 / 历史决策
- 技术可行性 / 风险评估

### 3. 设计
- 架构图（模块依赖）
- 数据流图（输入 → 处理 → 输出）
- 接口设计（API / 数据格式）

### 4. 任务拆解
- 单一职责子任务
- DAG 依赖图
- 每个子任务有 done 定义

### 5. 接力计划
- 哪个智能体负责哪部分
- AGENTS.md / WORKLOG.md / PROJECT_STATUS.md 同步
- 异常切换方案

### 6. 实现
- 渐进式（每步 < 200 行）
- 立即测试
- git commit 即时

### 7. 测试
- 单元测试（pytest）
- 集成测试（end-to-end）
- 手动验证（用户场景）

### 8. 部署
- 容器化（Docker）
- CI/CD（GitHub Actions）
- 监控（logs / metrics）

### 9. 监控
- 性能指标
- 错误日志
- 用户反馈

## 关键决策
- **渐进式** > 一次性（V1.0 → V1.1 → V1.2）
- **接力** > 单点（5 智能体可接手）
- **DAG** > 线性（任务依赖清晰）
- **文档** > 口口相传（WORKLOG / PROJECT_STATUS）
- **小工具** > 大框架（专用 > 通用）
