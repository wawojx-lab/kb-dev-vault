# 知识库架构调研

> 调研时间：2026-06-20
> 调研目的：为知识库架构调整提供行业方案支撑

## 调研结论

采用四层架构：格式层（OKF）+ 隔离层（双 vault）+ 写入层（所有智能体可写）+ 编译层（flywheel + Dynamic Workflows）

## 1. Google OKF（Open Knowledge Format）

**发布**：2026-06-16，Google Cloud
**核心**：把 Karpathy 的 LLM-wiki 模式标准化为可移植格式

### 格式
- Markdown 目录 + YAML frontmatter + Markdown 链接 = 知识图谱
- `type` 是唯一强制字段
- 概念之间用普通 Markdown 链接互相引用

### 三个设计原则
1. **最小约束**：只强制 type 字段，其他全自由
2. **生产者消费者独立**：人写的文件 AI 能读，AI 生成的 bundle 另一个 AI 能查
3. **格式不是平台**：不绑定云服务/SDK/账号，纯开放标准

### 对我们的启示
`_kb/wiki/` 采用 OKF 格式，确保未来可与其他工具互通。flywheel 本质就是 OKF 模式的实现。

### 参考链接
- https://github.com/GoogleCloudPlatform/knowledge-catalog/tree/main/okf

## 2. Claude Code Dynamic Workflows

**发布**：2026-05-28，Anthropic（Claude Code v2.1.154+）
**核心**：AI 临场写 JavaScript 编排脚本，后台跑，派生 subagent 并行干活

### 与 subagent/skill 的区别
| 方式 | 谁握计划 | 中间结果 | 上下文占用 |
|------|---------|---------|-----------|
| Subagent | Claude 逐轮判断 | 回到对话上下文 | 高 |
| Skill | Claude 按 SOP 走 | 回到对话上下文 | 高 |
| Dynamic Workflow | 脚本自己决定 | 在脚本变量里流转 | 低（只回最终结果） |

### 适合场景
- 代码库批量扫描（架构/安全/性能/测试四维并行）
- 大规模迁移（Bun 作者 11 天 75 万行 Zig→Rust）
- 研究类任务（/deep-research 内置命令）
- 内容生产（多 agent 各管一摊）
- 多角度决策（强制不同立场，最后必须给结论）

### 不适合
- 一两步小修补
- 需频繁拍板的探索
- 安全/支付高风险改动

### 对我们的启示
可用 Dynamic Workflows 做大规模知识库编译——多 agent 并行读源、交叉验证、合成 wiki，中间过程不占主上下文。

## 3. flywheel（LLM-wiki 知识编译引擎）

**本质**：OKF 模式的具体实现
**工具数**：28 个（ingest/compile/query/lint/evolve）
**环境要求**：Python 3.12+，uvx 启动
**与 obsidian MCP 关系**：不冲突，职责正交（obsidian 是文件 IO，flywheel 是知识编译）

### 安装方式
- clone 到 `_kb_flywheel/`，独立 venv
- `pip install -e .`（纯运行时）
- 设 `KB_PROJECT_ROOT` 环境变量

### 与现有系统的关系
| MCP | 职责 | 给谁装 |
|-----|------|--------|
| obsidian MCP | 文件级读写 vault | 所有智能体 |
| kb-mcp（flywheel） | 知识编译 28 工具 | 仅知识管家 |

## 综合方案：四层架构

| 层 | 职责 | 工具 | 谁来用 |
|----|------|------|--------|
| 格式层 | OKF 标准 | 无（纯规范） | 所有智能体写入时遵循 |
| 隔离层 | 双 vault 物理隔离 | obsidian MCP × 2 | 开发域：所有智能体；个人域：指定智能体 |
| 写入层 | 所有智能体可写入 | obsidian MCP | 所有智能体 |
| 编译层 | 知识编译/治理 | flywheel + Dynamic Workflows | 知识管家 |

## 落地步骤
1. ✅ 建 `_kb_personal/` 个人域 vault
2. ✅ `_kb/` 改造为 raw/wiki/research/projects 四层
3. ✅ 写 `_kb/README.md` 新写入规则
4. ✅ clone flywheel 到 `_kb_flywheel/`，隔离试跑 demo（lint 通过）
5. ✅ Claude Code 装 kb MCP（CLI 模式实测 27 个工具，官方文档 28 个）
6. ✅ 双 vault 隔离（kb MCP 写开发域 + 文件系统兜底写个人域）
7. ✅ 现有内容迁移（20-areas/ 已删，迁移到 wiki/）

> 写入权限说明：取消原"单点写入"限制，**所有智能体可写** raw/wiki/research/projects；flywheel 知识管家负责治理（lint/compile/evolve）
