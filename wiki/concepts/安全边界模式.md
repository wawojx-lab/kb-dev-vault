# Agent 安全边界模式（Agent Security Boundary Patterns）

AI Agent 部署到企业生产环境时，**必须强制执行的安全隔离层**。单纯依赖 Prompt 控制权限是反模式 — Agent 越狱、Prompt 注入、Token 泄漏都可能绕过。

## 安全五层模型

```
┌─────────────────────────────────────────────┐
│ L1: 网络层（Network）                        │
│   - VPC / PrivateLink / 安全组               │
│   - Agent 不暴露公网 IP                       │
├─────────────────────────────────────────────┤
│ L2: 身份层（Identity）                       │
│   - 用户身份（OAuth / SAML / OIDC）           │
│   - Agent 身份（Service Account）             │
│   - 跨服务委派（Token Exchange）              │
├─────────────────────────────────────────────┤
│ L3: 权限层（Authorization）                  │
│   - IAM Policy / RBAC / Cedar 策略          │
│   - 工具级 + 数据级 + 操作级                  │
├─────────────────────────────────────────────┤
│ L4: 加密层（Encryption）                     │
│   - KMS 静态加密                              │
│   - TLS 1.3 传输加密                          │
│   - 凭据托管（Secrets Manager）               │
├─────────────────────────────────────────────┤
│ L5: 审计层（Audit）                          │
│   - CloudTrail / OpenTelemetry               │
│   - 完整调用链追溯                            │
│   - 异常行为告警                              │
└─────────────────────────────────────────────┘
```

## 核心模式

### 模式 1：IAM 最小权限（Least Privilege）
Agent 每个工具调用都对应独立 IAM Role，权限按"业务动词"切分：

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": ["s3:GetObject"],
      "Resource": "arn:aws:s3:::company-public-docs/*"
    },
    {
      "Effect": "Deny",
      "Action": ["s3:Delete*", "s3:Put*"],
      "Resource": "*"
    }
  ]
}
```

**反模式**：给 Agent 配 AdministratorAccess。

### 模式 2：VPC + PrivateLink（网络隔离）
- Agent Runtime 部署在私有子网
- 访问 AWS 服务走 PrivateLink 端点（不出公网）
- 外部 API 走 NAT Gateway + 域名白名单

**关键指标**：**Agent 实例永不分配公网 IP**。

### 模式 3：KMS 加密 + Secrets 托管
- 所有 S3 / DynamoDB / EBS 数据用 Customer Managed Key (CMK) 加密
- 数据库密码、API Token 放 Secrets Manager，运行时动态拉取
- Agent 代码里**永不出现明文密钥**

```python
# 错
db_password = "my_secret_123"

# 对
import boto3
secret = boto3.client('secretsmanager').get_secret_value(SecretId='db/app')['SecretString']
```

### 模式 4：策略实时拦截（Policy Enforcement）
- AgentCore Policy 用 **Cedar 策略语言**实时拦截工具调用
- 策略可写为自然语言（"不允许删除生产数据库"）→ 自动转 Cedar

```cedar
forbid (
    principal,
    action == Action::"s3:DeleteObject",
    resource in S3::Bucket::"prod-financial-data"
);
```

**优势**：策略可热更新，不重启 Agent。

### 模式 5：完全会话隔离（Session Isolation）
- 每个 Agent 会话独立容器/微 VM
- 会话间不共享内存、文件系统、网络
- AgentCore Runtime 默认 8h 超时

**反模式**：多租户共用一个 Agent 进程。

### 模式 6：完整审计链（Audit Trail）
每次工具调用记录：

| 字段 | 示例 |
|------|------|
| trace_id | `01HXYZ...` |
| user_id | `u-12345` |
| agent_id | `agent-cfo-bot` |
| tool_name | `create_purchase_order` |
| params_hash | `sha256:abc...`（不存明文） |
| timestamp | `2026-06-20T12:34:56Z` |
| result | `success / error_code` |
| duration_ms | `342` |

**存储**：CloudTrail + 业务专用 audit log（S3 + Athena 查询）。

## 必须避免的反模式

| 反模式 | 风险 | 替代方案 |
|--------|------|----------|
| Prompt 写权限规则 | 越狱绕过 | Gateway 层 IAM 强制 |
| Agent 直接连生产 DB | 数据泄漏/误删 | API 网关 + 只读账号 |
| 共享 Admin 凭据 | 凭据泄漏 | 短期 STS Token |
| 工具返回原始 SQL | 注入攻击 | 参数化 + ORM |
| 日志记录用户 PII 明文 | 隐私合规违规 | 脱敏 + hash |
| 关闭 CloudTrail | 无法审计 | 全量开启 + 跨区复制 |

## 选型检查清单

部署 Agent 到生产前必须验证：

- [ ] Agent 是否有独立 IAM Role（非共享）
- [ ] 网络是否在 VPC 内（无公网 IP）
- [ ] 凭据是否从 Secrets Manager 拉取
- [ ] 是否配置了 KMS CMK 加密
- [ ] 是否启用了 CloudTrail 审计
- [ ] 是否设置 Policy 实时拦截
- [ ] 是否有异常调用告警（如短时间大量删除）
- [ ] 日志是否脱敏（PII / 密钥）
- [ ] 是否有会话超时和资源限制
- [ ] 是否有灾备切换方案（Region Failover）

## 实施代价 vs 安全等级

| 等级 | 措施 | 实施时间 | 适用 |
|------|------|----------|------|
| L1 基础 | IAM + Secrets + 日志 | 1-2 天 | POC |
| L2 标准 | + VPC + KMS + CloudTrail | 3-5 天 | 生产 |
| L3 严格 | + Policy + 会话隔离 + 告警 | 1-2 周 | 金融/医疗 |
| L4 极致 | + 多区容灾 + 端到端加密 + 零信任 | 1 月+ | 国企/政府 |
