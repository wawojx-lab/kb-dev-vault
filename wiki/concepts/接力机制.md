# 接力机制（Relay Mechanism）

跨 AI 智能体（Trae / Codex / Claude Code / Hermes / OpenCode）间传递上下文的协议。

## 核心原则
- **每次会话开始必读 3 件套**：
  - `AGENTS.md`（或 `CLAUDE.md`） — 项目规则
  - `WORKLOG.md` — 工作日志
  - `PROJECT_STATUS.md` — 项目状态
- **每次会话结束必写 2 件套**：
  - 更新 `WORKLOG.md`（加一条）
  - 更新 `PROJECT_STATUS.md`（推进状态）
- **统一入口**：`CURRENT_TASK.md` — 当前多步任务的状态

## 关键文件位置
- 项目级：`{项目}/AGENTS.md` + `WORKLOG.md` + `PROJECT_STATUS.md`
- 工作区级：`d:\xiangmu\WORKLOG.md` + `d:\xiangmu\CURRENT_TASK.md`
- 通用规范：`d:\xiangmu\_meta\AGENTS.md`

## 验证状态
- ✅ 5 智能体接力验证（Trae + Codex + Claude Code + Hermes + OpenCode，2026-06-20）
- ✅ stock-sim V1.2 接力流程走通

## 适用场景
- 跨会话：session 中断后下一会话接手
- 跨智能体：同一项目由不同 AI 工具协作
- 跨设备：本地 / 远程 同步
