# 渐进式开发（Incremental Development）

用户偏好的开发哲学："完成一个，记录一个"（Complete one, record one），避免长周期不反馈。

## 核心原则
- **小步快跑**：每次提交改动量 < 200 行
- **立即反馈**：完成即测试，测试即记录
- **可回退**：每个里程碑有可回退点
- **用户可见**：每完成一个功能用户能用上
- **优先级驱动**：先做"必要的"后做"锦上添花的"

## 实践方式
- **CHANGELOG.md**：每个版本记录"做了什么 + 为什么"
- **git tag**：每个里程碑打 tag（v0.5 / v1.0 / v1.2）
- **定期演示**：每完成一批功能向用户展示
- **失败快速识别**：跑测试 + 监控日志

## 适用场景
- 业务迭代（用户能用到新功能 → 反馈 → 改进）
- MVP 开发（先做最小可用版）
- 试点项目（50 股票跑通 + 验证 → 推 500 股票）

## 关键决策
- 用户在 stock-sim 中选择"50 股票 + 跑通 + 推 500"而非"500 股票 + 一次跑"
- V1.0 → V1.1 → V1.2 渐进式推进
